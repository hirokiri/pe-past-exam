# 分析: 著作権法上の「引用」（第32条）として利用する

## Source

- Issue: https://github.com/hirokiri/pe-past-exam/issues/12 (#12)
- Slug: `copyright-quotation`

## Goal

過去問題文の掲載を著作権法32条の「引用」の要件を満たす形に整える。具体的には
(1) 問題文（引用部分）を視覚的に明瞭区分し出所を明示する、
(2) topページで「解説・模範解答が主、過去問引用が従」という主従関係を明示する。
問題詳細ページで問題文が枠囲み＋出典付きで表示され、topページに主従関係の説明が載っていれば「done」。

## Requirements

Issue の「実装の方法」で明示された対応:

- [ ] 過去問題部分（`## 問題` セクション）を四角のボーダーで囲み、解説・模範解答と視覚的に明瞭区分する
- [ ] topページに主従関係（本サイトは模範解答・解説が主であり、過去問題文は引用＝従である旨）を明示する

Issue 本文の引用要件から導かれる付随要件:

- [ ] 出所の明示（第48条）: 引用枠内またはその直下に「出典：公益社団法人 日本技術士会 {yearLabel} {examLabel}（{divisionLabel}）{questionNo}」を frontmatter から自動生成して表示する（コンテンツ70ファイルの手修正はしない）
- [ ] 改変禁止・第三者著作物への配慮は運用ルールであり、本実装ではコード変更対象外（既存の転記フローで担保）

非機能:

- [ ] 既存70ファイルの Markdown 構造（`## 問題` → `## 模範解答` → `## 解説`）を変更せずに実現する（全70ファイルで `## 問題` 見出しの存在を確認済み）
- [ ] 既存テスト（`tests/content.test.ts`, `tests/search.test.ts`）を green に保つ
- [ ] KaTeX / Mermaid のクライアントレンダリング（`question-content` 内を走査）を壊さない

## Affected areas

- `app/routes/question.tsx` — 問題詳細ページ。loader が `renderMarkdown(body)` で本文全体を1つのHTMLにしている。問題セクションを分離して枠囲み＋出典表示するならここが中心（body を `## 模範解答` 以降と分割して2つのHTMLにする、等）。
- `app/lib/markdown.server.ts` — Markdown→HTML 変換。分割ヘルパーを置く候補。
- `app/app.css` — `question-content` 系のスタイル定義（223行目〜）。引用枠（四角ボーダー）と出典表記のスタイルを追加。既存の `--border` / `--card-bg` 等のCSS変数を使う。
- `app/routes/home.tsx` — topページ。hero セクションに主従関係の明示文を追加する。
- `app/root.tsx` — フッターに既に「過去問題の著作権は公益社団法人日本技術士会に帰属します。本サイトは非公開・私的利用に限定しています。」があり、これは維持。
- （周辺・任意）`scripts/export.ts` — Vivliostyle での PDF/EPUB 書き出しは Markdown をそのまま原稿化しており、Web 表示とは別経路。Issue は Web サイト表示を対象としているため本対応のスコープ外とする（下記 assumptions）。

## Constraints & assumptions

- **区分方法**: 全70ファイルが `## 問題` 見出しで始まり、次の `## `（模範解答）までが問題文。Markdown 側を書き換えず、サーバー側で本文を分割して問題部分を枠要素（`<blockquote>` 相当のクラス付きコンテナ）で囲む方式が、既存コンテンツ非破壊で最も安全。
- **出典リンク**: frontmatter の `sourcePdf` はリポジトリ内ローカルパス（例 `data/pdf/second/01-kikai/r01_0103.pdf`）で公開URLではない。日本技術士会の問題別公開URLは frontmatter に持っていないため、リンクは張らずテキスト出典表記のみとする（Issue も「可能であれば」の位置づけ）。
- **主従関係の文言**: topページ hero に、「本サイトは模範解答・解説の提供を目的とし（主）、問題文は著作権法第32条に基づく引用として必要な範囲で掲載しています（従）。出典は各ページに明示しています。」という趣旨の短い説明を追加する。文言の細部は実装時に調整。
- **改変禁止・第三者著作物（図の省略等）**: コンテンツ作成時の運用ルールであり、本 Issue のコード実装対象は上記2点＋出典表示に限定。
- サイトは現状非公開・私的利用前提（フッター表記）だが、本対応は公開に備えた引用要件整備と解釈する。

## Acceptance criteria

1. 問題詳細ページ（`/questions/:id`）で、`## 問題` セクション全体が四角いボーダーの枠で囲まれて表示され、模範解答・解説は枠外に表示される。
2. 枠内または枠直下に「出典：公益社団法人 日本技術士会 {年度} {試験名}（{部門}）{問題番号}」形式の出所明示が全問題で自動表示される。
3. topページに主従関係（解説・模範解答が主、過去問引用が従、著作権法32条に基づく引用である旨）の説明が表示される。
4. KaTeX 数式・Mermaid 図が問題枠の内外どちらでも従来どおりレンダリングされる（`contentRef` の走査対象に問題部分が含まれること）。
5. `bun test`（content/search テスト）と `bun run build`（型チェック含む）が green。
6. content/ 配下の Markdown 70ファイルは変更しない。
